## 支持的 AI Provider

SSH Terminal 支持多种 AI 服务提供商，可以根据需求选择：

### OpenAI

- **支持的模型**：GPT-4、GPT-3.5-turbo、GPT-4-turbo 等
- **特点**：
  - 强大的代码理解和生成能力
  - 支持流式输出
  - 上下文长度可达 128K tokens
- **配置要求**：
  - API Key
  - 可选：自定义 Base URL（使用代理）

### Ollama

- **支持的模型**：Llama 2、Mistral、Qwen 等本地模型
- **特点**：
  - 完全本地运行，保护隐私
  - 无需 API 费用
  - 支持自定义模型
- **配置要求**：
  - Ollama 服务地址（默认：http://localhost:11434）
  - 模型名称

### Qwen (通义千问)

- **支持的模型**：qwen-turbo、qwen-plus、qwen-max 等
- **特点**：
  - 阿里云提供，中文能力强
  - 价格实惠
  - 支持长上下文
- **配置要求**：
  - DashScope API Key
  - 模型名称

### Wenxin (文心一言)

- **支持的模型**：ERNIE-Bot、ERNIE-Bot-turbo 等
- **特点**：
  - 百度提供，中文优化
  - 理解能力强
  - 价格合理
- **配置要求**：
  - 百度云 API Key 和 Secret Key

### 其他兼容 OpenAI API 的服务

- DeepSeek、Moonshot、Zhipu 等
- 只需提供兼容 OpenAI API 格式的接口

## AI 功能模块

### 1. 自然语言转命令 (NL to Command)

**功能描述**：用自然语言描述需求，AI 自动生成对应的 Shell 命令。

**使用方法**：

1. **快捷键触发**
   - Windows/Linux: `Ctrl+Shift+A`
   - macOS: `Cmd+Shift+A`

2. **点击按钮**
   - 在终端工具栏点击 "AI 转命令" 按钮

3. **输入需求**
   - 在弹出的对话框中输入自然语言描述
   - 示例：
     - "查看当前目录下所有文件"
     - "查找所有大于 100MB 的文件"
     - "统计当前目录下文件数量"
     - "压缩当前目录为 zip 文件"

4. **获取命令**
   - AI 生成对应的 Shell 命令
   - 显示命令和简要说明
   - 可选择：
     - 插入到终端
     - 复制到剪贴板
     - 重新生成

**技术实现**：

```typescript
// 用户输入自然语言
const userInput = "查看当前目录下所有文件";

// 构建提示词
const prompt = `
你是一个 Linux 命令行专家。请根据用户的自然语言描述，生成对应的 Shell 命令。

用户需求：${userInput}

请返回以下格式的 JSON：
{
  "command": "ls -la",
  "explanation": "ls -la 会列出当前目录下的所有文件，包括隐藏文件，并显示详细信息"
}
`;

// 调用 AI API
const response = await aiProvider.chat(prompt, {
  temperature: 0.3,
  max_tokens: 500
});

// 解析并显示命令
const result = JSON.parse(response.content);
```

**最佳实践**：

- 描述要清晰具体
- 说明目标目录或文件
- 指定操作类型（查看、修改、删除等）
- 如有特殊要求，请在描述中说明

### 2. 命令解释器

**功能描述**：选中终端中的命令，AI 提供详细解释和用法说明。

**使用方法**：

1. **选中命令**
   - 在终端中用鼠标选中要解释的命令
   - 或使用快捷键选中整行

2. **触发解释**
   - 右键菜单选择 "AI 解释命令"
   - 或使用快捷键（可自定义）

3. **查看解释**
   - AI 返回详细的命令解释
   - 包括：
     - 命令作用
     - 参数说明
     - 使用示例
     - 注意事项

**示例**：

输入命令：`find /var/log -name "*.log" -mtime +7 -size +100M`

AI 解释：

```
【命令作用】
查找 /var/log 目录下 7 天前修改的、大小超过 100MB 的 .log 文件

【参数说明】
- /var/log：搜索的目录路径
- -name "*.log"：匹配文件名以 .log 结尾的文件
- -mtime +7：修改时间超过 7 天
- -size +100M：文件大小超过 100MB

【使用示例】
1. 查找并列出符合条件的文件：
   find /var/log -name "*.log" -mtime +7 -size +100M -ls

2. 删除符合条件的文件：
   find /var/log -name "*.log" -mtime +7 -size +100M -delete

【注意事项】
- 使用 -delete 参数前，建议先用 -ls 或 -print 查看匹配的文件
- find 命令会递归搜索子目录
- 大型目录搜索可能耗时较长
```

### 3. 错误分析器

**功能描述**：当终端出现错误时，选中错误信息，AI 分析错误原因并提供解决方案。

**使用方法**：

1. **选中错误信息**
   - 在终端中选中错误消息
   - 包括错误代码和相关信息

2. **触发分析**
   - 右键菜单选择 "AI 分析错误"
   - 或使用快捷键（可自定义）

3. **查看分析结果**
   - AI 分析错误原因
   - 提供可能的原因
   - 给出解决方案
   - 提供预防建议

**示例**：

错误信息：`Permission denied: /var/www/html/config.php`

AI 分析：

```
【错误原因】
当前用户没有权限访问 /var/www/html/config.php 文件

【可能的原因】
1. 文件所有者不是当前用户
2. 文件权限设置过于严格
3. 当前用户不在允许的用户组中

【解决方案】

方案 1：使用 sudo 提升权限（谨慎使用）
sudo cat /var/www/html/config.php

方案 2：修改文件权限
sudo chmod 644 /var/www/html/config.php

方案 3：修改文件所有者
sudo chown $USER:$USER /var/www/html/config.php

【预防建议】
- 定期检查文件权限设置
- 使用适当的用户组管理文件访问
- 谨慎使用 sudo，了解其风险
```

**功能描述**：独立的全屏聊天界面，支持多轮对话和上下文理解。

**使用方法**：

1. **打开 AI 聊天**

3. **功能特性**

   - 支持代码高亮显示

   - 支持导出对话记录

**对话历史管理**：

- 每个服务器的对话独立管理
- 按时间排序
- 支持搜索历史对话
- 支持删除对话
- 支持导出对话（Markdown 格式）

## AI 配置

### 配置入口

1. 打开设置页面
2. 进入 "AI" 标签
3. 配置 AI Provider

### 基本配置

**选择 AI Provider**

从下拉列表中选择要使用的 AI 服务商。

**配置 API Key**

- OpenAI: 输入 OpenAI API Key
- Qwen: 输入 DashScope API Key
- Wenxin: 输入百度云 API Key 和 Secret Key
- Ollama: 配置服务地址（默认：http://localhost:11434）

**选择模型**

根据选择的 Provider，选择对应的模型：
- OpenAI: gpt-4、gpt-3.5-turbo 等
- Qwen: qwen-turbo、qwen-plus、qwen-max 等
- Wenxin: ERNIE-Bot、ERNIE-Bot-turbo 等
- Ollama: 已安装的模型名称

### 高级配置

**温度 (Temperature)**

- 控制输出随机性
- 范围：0.0 - 2.0
- 推荐值：
  - 代码生成：0.0 - 0.3
  - 命令解释：0.3 - 0.5
  - 对话：0.5 - 1.0

**最大 Tokens (Max Tokens)**

- 控制响应最大长度
- 范围：100 - 4096
- 推荐值：
  - 命令生成：500 - 1000
  - 命令解释：1000 - 2000
  - 对话：2000 - 4096

**Top P**

- 控制采样范围
- 范围：0.0 - 1.0
- 推荐值：0.9 - 1.0

**系统提示词 (System Prompt)**

自定义 AI 的行为和风格：

```
你是一个 Linux 命令行专家，擅长 Shell 脚本和系统管理。
请提供准确、简洁的命令和解释。
```

### 自定义 Base URL

如果需要使用代理或自定义 API 端点，可以配置 Base URL：

- OpenAI 兼容服务：输入自定义 API 地址
- 示例：`https://api.openai-proxy.com/v1`

## AI 缓存机制

SSH Terminal 实现了智能的 AI 缓存机制，提高响应速度，减少 API 调用次数。

### 缓存策略

**1. 请求缓存**

- 相同的请求会返回缓存结果
- 缓存时间：30 分钟
- 缓存大小：100 条

**2. Provider 缓存池**

- 多个 AI Provider 负载均衡
- 智能选择响应最快的 Provider
- 自动故障转移

**3. 流式响应缓存**

- 流式响应实时显示
- 完整响应缓存到本地
- 支持离线查看历史对话

### 缓存管理

**查看缓存**

在设置页面的 AI 标签中，可以查看：
- 缓存命中率
- 缓存条数
- 缓存大小

**清除缓存**

可以手动清除缓存：
- 清除特定对话的缓存
- 清除所有缓存

## 最佳实践

### 1. 提示词优化

**描述要清晰具体**

- ❌ "查看文件"
- ✅ "查看 /var/log 目录下今天修改的 .log 文件"

**说明上下文**

- ❌ "删除文件"
- ✅ "删除 /tmp 目录下所有 .tmp 文件"

**指定输出格式**

- ❌ "统计文件"
- ✅ "统计当前目录下文件数量，只返回数字"

### 2. 温度设置

**代码生成**：使用较低的温度（0.0 - 0.3）
- 生成稳定的代码
- 减少随机性

**对话交流**：使用中等的温度（0.5 - 0.8）
- 更自然的对话
- 更多的创意

**创意写作**：使用较高的温度（0.8 - 1.0）
- 更有创意的输出
- 多样化的回答

### 3. 安全考虑

**不要输入敏感信息**
- 不要输入密码、密钥等敏感信息
- AI 提示词可能被记录

**验证生成的命令**
- AI 生成的命令需要验证
- 特别是有破坏性操作的命令
- 建议先在测试环境验证

**使用可信的 AI Provider**
- 选择正规的 AI 服务商
- 避免使用不明来源的 API

## 常见问题

### Q: 为什么 AI 响应很慢？

A: 可能的原因：
1. 网络问题：检查网络连接
2. API 限流：检查 API 使用限制
3. 模型参数：减少 max_tokens 可以提高速度
4. 缓存未命中：相同请求会更快

### Q: AI 生成的命令不正确怎么办？

A:
1. 尝试更详细的描述
2. 指定上下文和目标
3. 使用命令解释功能学习正确的用法
4. 在 AI 聊天中提问获取帮助

### Q: 如何离线使用 AI？

A:
1. 使用 Ollama 本地模型
2. 配置 Ollama 服务地址
3. 选择本地模型
4. 完全离线运行

### Q: AI 对话历史会保存多久？

A:
- 对话历史保存在本地数据库
- 默认保存 30 天
- 可以在设置中修改保留时间
- 可以手动删除对话
