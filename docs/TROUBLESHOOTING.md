## 安装和启动问题

### Q: 安装依赖失败

**问题**：运行 `pnpm install` 时报错

**可能原因**：
1. Node.js 版本不符合要求
2. 网络问题
3. pnpm 版本过低

**解决方案**：

```bash
# 1. 检查 Node.js 版本（需要 18+）
node --version

# 2. 检查 pnpm 版本（需要 10.14.0+）
pnpm --version

# 3. 更新 pnpm
npm install -g pnpm@latest

# 4. 清除缓存后重新安装
pnpm store prune
rm -rf node_modules pnpm-lock.yaml
pnpm install
```

### Q: Rust 编译失败

**问题**：运行 `pnpm tauri dev` 时 Rust 编译失败

**可能原因**：
1. Rust 版本不符合要求（需要 1.70+）
2. 系统依赖缺失
3. Cargo 缓存损坏

**解决方案**：

```bash
# 1. 检查 Rust 版本
rustc --version

# 2. 更新 Rust
rustup update

# 3. 清理 Cargo 缓存
cd src-tauri
cargo clean

# 4. 重新构建
cargo build
```

**Windows 特定问题**：

如果遇到链接错误，可能需要安装 Microsoft C++ Build Tools：

1. 下载 [Microsoft C++ Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/)
2. 安装时选择 "Desktop development with C++"
3. 重新构建

### Q: 应用启动失败

**问题**：应用无法启动，闪退或报错

**解决方案**：

```bash
# 1. 查看错误日志
pnpm tauri dev

# 2. 检查端口是否被占用（1420）
netstat -ano | findstr :1420  # Windows
lsof -i :1420                  # macOS/Linux

# 3. 清除应用数据
# Windows: %APPDATA%\ssh-terminal
# macOS: ~/Library/Application Support/ssh-terminal
# Linux: ~/.config/ssh-terminal
```

### Q: 热更新不工作

**问题**：修改代码后应用不自动刷新

**解决方案**：

```bash
# 1. 检查 Vite 开发服务器是否运行
pnpm dev

# 2. 检查端口 1420 是否被占用

# 3. 重启开发服务器
Ctrl+C
pnpm tauri dev

# 4. 清除浏览器缓存
```

## SSH 连接问题

### Q: SSH 连接失败

**问题**：点击连接后显示连接失败

**可能原因**：
1. 主机地址或端口错误
2. 用户名或密码错误
3. 网络问题
4. SSH 服务未启动

**解决方案**：

1. **检查连接信息**：
   - 确认主机地址、端口、用户名正确
   - 尝试使用其他 SSH 客户端连接（如 PuTTY、OpenSSH）

2. **检查网络**：
   ```bash
   # Ping 主机
   ping example.com

   # 检查端口是否开放
   telnet example.com 22
   ```

3. **检查 SSH 服务**：
   ```bash
   # 在服务器上检查 SSH 服务状态
   systemctl status sshd  # Linux
   service ssh status     # Ubuntu/Debian
   ```

4. **查看详细错误**：
   - 打开开发者工具（F12）
   - 查看控制台错误信息

### Q: 认证失败

**问题**：输入密码后认证失败

**可能原因**：
1. 密码错误
2. 不支持密码认证
3. 密钥认证配置错误

**解决方案**：

1. **使用密钥认证**：
   - 生成 SSH 密钥对
   - 将公钥添加到服务器
   - 在应用中配置私钥

2. **检查密码**：
   - 确认密码正确
   - 注意大小写和特殊字符

3. **查看服务器日志**：
   ```bash
   # 查看服务器 SSH 日志
   tail -f /var/log/auth.log  # Ubuntu/Debian
   tail -f /var/log/secure    # CentOS/RHEL
   ```

### Q: 主机密钥验证失败

**问题**：连接时提示主机密钥不匹配

**解决方案**：

1. **验证主机密钥**：
   - 联系服务器管理员确认密钥变化
   - 如果服务器已更新，接受新密钥

2. **清除 known_hosts**：
   ```bash
   # 删除特定主机的记录
   ssh-keygen -R example.com

   # 或删除整个文件
   rm known_hosts
   ```

3. **在应用中重新连接**：
   - 应用会提示接受新密钥
   - 确认后保存

### Q: 连接频繁断开

**问题**：SSH 连接经常自动断开

**解决方案**：

1. **配置心跳保活**：
   - 在会话设置中启用心跳
   - 设置心跳间隔（推荐 30-60 秒）

2. **检查网络稳定性**：
   - 测试网络连接质量
   - 使用 VPN 或更稳定的网络

3. **调整服务器配置**：
   ```bash
   # 编辑服务器 SSH 配置
   sudo vim /etc/ssh/sshd_config

   # 添加以下配置
   ClientAliveInterval 60
   ClientAliveCountMax 3

   # 重启 SSH 服务
   sudo systemctl restart sshd
   ```

## 终端显示问题

### Q: 终端输出乱码

**问题**：终端显示的字符是乱码

**可能原因**：
1. 编码设置不正确
2. 服务器使用非 UTF-8 编码

**解决方案**：

1. **检查终端编码**：
   - 在应用设置中检查编码设置
   - 默认使用 UTF-8

2. **在服务器上设置编码**：
   ```bash
   # 添加到 ~/.bashrc 或 ~/.zshrc
   export LANG=en_US.UTF-8
   export LC_ALL=en_US.UTF-8

   # 重新加载配置
   source ~/.bashrc
   ```

3. **使用 locale 命令**：
   ```bash
   # 查看当前 locale
   locale

   # 设置 locale
   export LC_CTYPE=en_US.UTF-8
   ```

### Q: 终端显示异常

**问题**：终端显示不正常，如错位、重叠等

**解决方案**：

1. **重置终端**：
   ```bash
   reset
   ```

2. **调整终端尺寸**：
   - 在应用中调整终端窗口大小
   - 确保服务器识别到正确的终端尺寸

3. **检查 TERM 环境变量**：
   ```bash
   # 查看 TERM 变量
   echo $TERM

   # 设置 TERM 变量
   export TERM=xterm-256color
   ```

### Q: 终端无响应

**问题**：终端卡住，无法输入或显示输出

**解决方案**：

1. **检查连接状态**：
   - 确认 SSH 连接仍然活跃
   - 尝试发送心跳

2. **使用快捷键**：
   - `Ctrl+C` - 发送中断信号
   - `Ctrl+Z` - 挂起进程
   - `Ctrl+D` - 发送 EOF

3. **重新连接**：
   - 关闭当前终端标签
   - 重新连接

## AI 功能问题

### Q: AI 响应很慢

**问题**：AI 生成响应需要很长时间

**解决方案**：

1. **检查网络**：
   - 确认网络连接正常
   - 测试到 AI API 的延迟

2. **调整 AI 参数**：
   - 减少 `max_tokens` 值
   - 降低 `temperature` 值
   - 选择更快的模型

3. **检查 API 配额**：
   - 确认 API Key 有效
   - 检查是否有使用限制

4. **使用缓存**：
   - 相同的请求会使用缓存
   - 缓存时间：30 分钟

### Q: AI 生成错误的结果

**问题**：AI 生成的命令或解释不正确

**解决方案**：

1. **优化提示词**：
   - 提供更详细的描述
   - 说明上下文和目标
   - 指定输出格式

2. **调整 AI 参数**：
   - 降低 `temperature` 值（更确定性）
   - 增加 `max_tokens` 值（更详细）

3. **使用不同的模型**：
   - 尝试使用更强大的模型
   - 如 GPT-4 而不是 GPT-3.5

4. **验证结果**：
   - AI 生成的命令需要验证
   - 特别是有破坏性操作的命令

### Q: AI 功能不可用

**问题**：AI 按钮或菜单无法点击

**可能原因**：
1. 未配置 AI Provider
2. API Key 无效
3. 网络问题

**解决方案**：

1. **配置 AI Provider**：
   - 打开设置
   - 进入 AI 标签
   - 选择 AI Provider
   - 输入 API Key

2. **验证 API Key**：
   - 确认 API Key 正确
   - 检查 API Key 是否过期

3. **检查网络**：
   - 确认可以访问 AI API
   - 测试网络连接

## 录制功能问题

### Q: 录制时 CPU 占用很高

**问题**：录制时应用卡顿，CPU 占用率高

**解决方案**：

1. **降低录制质量**：
   - 使用 Low 质量而不是 High
   - 降低帧率（30 FPS → 15 FPS）

2. **关闭音频录制**：
   - 如果不需要音频，关闭音频录制

3. **关闭其他程序**：
   - 关闭占用 CPU 的其他程序
   - 减少系统负载

### Q: 录制的视频文件太大

**问题**：录制几分钟的视频文件就有几百 MB

**解决方案**：

1. **使用更低的录制质量**：
   - 使用 Low 质量
   - 降低视频码率

2. **缩短录制时间**：
   - 分段录制
   - 只录制必要的部分

3. **录制后压缩**：
   - 使用视频压缩工具
   - 如 HandBrake、FFmpeg

### Q: 音频录制没有声音

**问题**：录制的视频没有声音

**解决方案**：

1. **检查麦克风权限**：
   - 确认应用有麦克风权限
   - 在系统设置中授予权限

2. **检查音频设置**：
   - 在应用设置中检查音频配置
   - 确认麦克风正常工作

3. **测试麦克风**：
   ```bash
   # 测试麦克风（macOS）
   afplay /System/Library/Sounds/Ping.aiff

   # 测试麦克风（Linux）
   arecord -d 5 test.wav
   aplay test.wav
   ```

### Q: 录制无法保存

**问题**：点击停止录制后，录制文件没有保存

**解决方案**：

1. **检查存储空间**：
   - 确认磁盘有足够空间
   - 清理不必要的文件

2. **检查写入权限**：
   - 确认应用有写入权限
   - 检查目标文件夹权限

3. **查看错误日志**：
   - 打开开发者工具
   - 查看控制台错误信息

## 同步功能问题

### Q: 同步失败

**问题**：点击同步后显示同步失败

**可能原因**：
1. 网络问题
2. 服务器不可用
3. 认证失败

**解决方案**：

1. **检查网络**：
   - 确认网络连接正常
   - 测试到服务器的连接

2. **检查服务器状态**：
   - 确认服务器正在运行
   - 检查服务器日志

3. **检查认证**：
   - 确认已登录
   - 检查 Token 是否有效
   - 尝试重新登录

### Q: 同步冲突

**问题**：同步时提示有冲突

**解决方案**：

1. **查看冲突详情**：
   - 应用会显示冲突的数据
   - 查看哪些数据有冲突

2. **选择解决策略**：
   - 保留服务器版本
   - 保留本地版本
   - 保留两个版本

3. **手动解决**：
   - 编辑冲突的数据
   - 重新同步

### Q: 数据丢失

**问题**：同步后本地数据丢失

**解决方案**：

1. **检查同步设置**：
   - 确认同步策略正确
   - 检查是否选择了错误的策略

2. **从服务器恢复**：
   - 如果数据在服务器上，可以恢复
   - 使用历史版本（如果有）

3. **检查备份**：
   - 查看是否有本地备份
   - 恢复备份数据

## 性能问题

### Q: 应用启动慢

**问题**：应用启动需要很长时间

**解决方案**：

1. **清理缓存**：
   ```bash
   # 清除应用缓存
   rm -rf ~/.config/ssh-terminal/cache
   ```

2. **减少启动项**：
   - 关闭不必要的自动连接
   - 减少加载的会话数量

3. **更新应用**：
   - 更新到最新版本
   - 性能优化可能在最新版本中

### Q: 应用运行卡顿

**问题**：应用运行时不流畅，卡顿

**解决方案**：

1. **关闭不必要的标签**：
   - 关闭不用的终端标签
   - 减少内存占用

2. **降低终端质量**：
   - 降低字体渲染质量
   - 减少终端缓冲区大小

3. **检查系统资源**：
   - 检查 CPU 和内存使用情况
   - 关闭其他占用资源的程序

4. **更新应用**：
   - 更新到最新版本
   - 性能优化可能在最新版本中
