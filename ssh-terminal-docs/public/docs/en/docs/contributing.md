# Contributing Guide

Welcome to contribute to the SSH Terminal project! This section will guide you on how to make contributions to the project.

## 📖 Contribution Flow

1. **Environment Setup** - Set up development environment following [Environment Setup](/docs/contributing/setup) guide
2. **Choose Task** - Select an issue you want to solve in Issues
3. **Create Branch** - Create a feature branch from master branch
4. **Write Code** - Write code following [Code Style](/docs/contributing/code-style)
5. **Run Tests** - Ensure tests pass following [Testing Guide](/docs/contributing/testing)
6. **Submit PR** - Submit Pull Request following [PR Flow](/docs/contributing/pr-flow)

## 💡 Contribution Suggestions

- **Start Small**: Start with simple bug fixes or documentation improvements
- **Communicate First**: Discuss with the community before starting work
- **Follow Standards**: Strictly follow project code style and standards
- **Write Tests**: Write corresponding tests for new features
- **Update Documentation**: Update documentation synchronously when code changes

## 🤝 Community Support

- **GitHub Discussions**: [Join Discussions](https://github.com/shenjianZ/ssh-terminal/discussions)
- **GitHub Issues**: [Submit Issues](https://github.com/shenjianZ/ssh-terminal/issues)
- **Pull Requests**: [Submit Code](https://github.com/shenjianZ/ssh-terminal/pulls)
