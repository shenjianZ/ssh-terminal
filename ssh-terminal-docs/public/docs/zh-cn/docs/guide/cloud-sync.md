# 云同步

SSH Terminal 提供强大的云同步功能，让你可以在多设备之间同步 SSH 会话、配置和用户资料。

---

## 云同步简介

云同步功能可以将你的重要数据安全地存储在云端，实现：

- **多设备访问** - 在不同设备上访问相同的配置
- **数据备份** - 自动备份重要数据，防止丢失

### 同步的数据类型

SSH Terminal 支持同步以下数据：

- **SSH 会话配置** - 所有保存的 SSH 连接配置
- **用户资料** - 用户名、头像等个人信息

**注意：** 密码和密钥等敏感信息经过加密后存储，确保安全。

---

## 配置云同步

### 账号注册

首次使用云同步需要注册账号：

1. 点击标题栏的头像图标，之后点击下方的注册
2. 填写注册信息：
   - **邮箱** - 用于登录和接收通知
   - **验证码（可选）** 用于验证邮箱注册
   - **密码** - 登录密码
3. 点击"注册"完成账号创建

**注意事项：**
- 密码需要满足强度要求（至少 8 位，包含字母和数字）
- 邮箱需要验证后才能使用所有功能

### 账号登录

已有账号可以直接登录：

1. 点击标题栏的头像图标
2. 输入邮箱和密码
3. 点击"登录"按钮

- 登录后会保存登录状态
- 下次启动应用时自动登录
- 点击标题栏头像，出现下拉菜单，点击"登出" 即可退出登录

### 找回密码（暂不支持）

如果忘记密码，可以重置：

1. 在登录页面点击"忘记密码"
2. 输入注册邮箱
3. 查收邮件中的重置链接
4. 按照邮件指引设置新密码

## 同步操作

### 手动同步

手动触发数据同步：

1. 在登录之后，点击标题栏的”同步“图标钮
2. 等待同步完成

**同步过程：**
- 上传本地修改的数据到云端
- 下载云端的数据到本地
- 合并两端的数据

### 自动同步（暂不支持）

启用自动同步后，应用会在以下情况自动同步：

- 配置更改后
- 定期检查（默认每 30 分钟）
- 应用启动时

**启用自动同步：**
1. 打开设置页面，切换到"同步"标签
2. 开启"自动同步"开关

**同步间隔设置：**
- 可以自定义同步间隔（15-120 分钟）
- 建议值：30 分钟

### 同步状态（仅支持查看最后同步时间）

在同步标签页可以查看：
- **最后同步时间** - 最近一次同步的时间
- **待同步数量** - 等待同步的数据项数量
- **冲突数量** - 需要解决的冲突数量
- **同步状态** - 当前同步状态（同步中、已完成、失败）

## 冲突处理

当本地和云端的数据不一致时，会产生冲突。

### 冲突类型

常见冲突场景：
- 同一会话在多个设备上同时修改
- 删除和修改操作冲突
- 配置版本不一致

### 冲突策略（目前服务器采用远程优先，且不支持配置）

SSH Terminal 提供三种冲突解决策略：

#### 本地优先
保留本地版本，覆盖云端数据

**适用场景：**
- 确定本地版本是正确的
- 网络中断后恢复同步

#### 远程优先
保留云端版本，覆盖本地数据

**适用场景：**
- 确定云端版本是正确的
- 从云端恢复数据

#### 保留最新
保留修改时间最新的版本

**适用场景：**
- 不确定哪个版本正确
- 希望保留最新的修改

### 解决冲突（目前服务器采用远程优先，且不支持配置）

当检测到冲突时：

1. 在同步标签页会显示冲突列表
2. 点击冲突项查看详细信息
3. 选择解决策略（本地优先 / 远程优先 / 保留最新）
4. 点击"解决"按钮
5. 冲突解决后自动重新同步

**批量解决：**
- 可以选择多个冲突项
- 应用相同的策略批量解决

## 常见问题

### 同步失败

可能原因：
- 网络连接中断
- 服务器维护
- 认证令牌过期

解决方案：
- 检查网络连接
- 重新登录账号
- 稍后重试

### 数据丢失

如果发现数据丢失：

1. 检查其他设备是否有数据
2. 查看同步历史，找到丢失的时间点
3. 从云端恢复数据
4. 联系技术支持

### 冲突频繁

如果频繁出现冲突：

- 避免同时在多个设备上修改同一配置
- 在修改前先同步最新数据
- 使用"保留最新"策略自动解决
