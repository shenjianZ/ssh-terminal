# SFTP 文件管理

SSH Terminal 集成了强大的 SFTP 文件管理功能，让你可以高效地管理远程服务器上的文件。

---

## SFTP 管理器简介

SFTP（SSH File Transfer Protocol）是一种安全的文件传输协议，通过 SSH 连接进行文件传输。

**主要功能：**
- 双面板设计，直观的文件浏览
- 支持上传和下载文件、文件夹
- 支持删除、重命名

---

## 界面布局

SFTP 管理器采用双面板设计：

### 左侧面板 - 本地文件
显示你本地电脑的文件系统，可以：
- 浏览本地文件夹
- 选择要上传的文件

### 右侧面板 - 远程文件
显示远程服务器上的文件系统，可以：
- 浏览远程文件夹
- 选择要下载的文件
- 管理远程文件

### 中间工具栏
提供常用操作的快捷按钮：
- 上传（将本地文件传输到服务器）
- 下载（将远程文件传输到本地）

---

## 基本操作

### 打开 SFTP 管理器

有两种方式可以打开 SFTP 管理器：
1. 点击侧边栏的"文件管理"图标
2. 在终端标签页右键，选择"打开 SFTP"

**注意：** 需要先建立 SSH 连接才能使用 SFTP 功能。

### 浏览文件

#### 本地文件
- 双击文件夹进入子目录
- 在上方路径栏快速导航

#### 远程文件
- 双击文件夹进入子目录
- 在上方路径栏快速导航

### 上传文件

将本地文件传输到远程服务器：

**方法一：使用工具栏**
1. 在左侧面板选择要上传的文件
2. 点击工具栏的"上传"按钮
3. 等待传输完成

**方法二：使用快捷键**
1. 选中要上传的文件
2. 按 `Ctrl+U` 快捷键

**批量上传：**
- 可以同时选中多个文件
- 可以选中整个文件夹上传

### 下载文件

将远程文件传输到本地电脑：

**方法一：使用工具栏**
1. 在右侧面板选择要下载的文件
2. 点击工具栏的"下载"按钮
4. 等待传输完成

**方法二：使用快捷键**
1. 选中要下载的文件
2. 按 `Ctrl+D` 快捷键

**批量下载：**
- 可以同时选中多个文件
- 可以选中整个文件夹下载

### 创建文件夹

在本远程面板创建新文件夹：

1. 点击工具栏的"新建文件夹"按钮
2. 输入文件夹名称
3. 按回车确认

### 删除文件

删除远程文件：

1. 选中要删除的文件或文件夹
2. 点击工具栏的"删除"按钮
3. 确认删除操作

**注意：** 删除操作不可恢复，请谨慎操作。

### 重命名文件

重命名远程文件：

1. 选中要重命名的文件
2. 点击工具栏的"重命名"按钮
3. 输入新名称
4. 按回车确认

### 刷新文件列表

如果文件列表发生变化，可以手动刷新：

1. 点击工具栏的"刷新"按钮
2. 或按 `F5` 快捷键

## 文件权限管理

### 查看文件权限

文件列表中显示文件的权限信息：
- `r` - 读取权限
- `w` - 写入权限
- `x` - 执行权限

格式示例：`-rwxr-xr-x`

---

## 快捷键

SFTP 管理器提供以下快捷键：

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| `Ctrl+U` | 上传文件 | 上传选中的本地文件到远程服务器 |
| `Ctrl+D` | 下载文件 | 下载选中的远程文件到本地 |
| `F5` | 刷新文件列表 | 刷新当前文件列表 |

**设计原则：**
- 使用 `F5` 而不是 `Ctrl+R`，避免与终端的反向搜索功能冲突
- 快捷键只在 SFTP 页面生效，不会影响其他功能

## 常见问题

### 上传失败

可能原因：
- 远程服务器磁盘空间不足
- 文件权限不足
- 网络连接中断

解决方案：
- 检查远程服务器磁盘空间
- 检查目标目录的写入权限
- 检查网络连接

### 下载失败

可能原因：
- 本地磁盘空间不足
- 远程文件不存在
- 网络连接中断

解决方案：
- 检查本地磁盘空间
- 确认远程文件存在
- 检查网络连接

### 无法浏览远程文件

可能原因：
- SSH 连接断开
- 远程目录权限不足
- SFTP 服务未启用

解决方案：
- 确认 SSH 连接正常
- 检查远程目录的读取权限
- 联系服务器管理员确认 SFTP 服务状态

### 传输速度慢

可能原因：
- 网络带宽限制
- 服务器负载高
- 文件过大

解决方案：
- 检查网络连接
- 选择网络空闲时段传输
- 分批传输大文件

## 安全性

SSH Terminal 的 SFTP 功能具有以下安全特性：

- **加密传输** - 所有文件传输都通过 SSH 加密
- **认证机制** - 使用 SSH 连接的认证方式
- **权限控制** - 遵循服务器文件系统的权限设置
