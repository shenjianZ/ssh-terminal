# 贡献者指南

欢迎参与 SSH Terminal 项目的贡献！本部分将指导你如何为项目做出贡献。


## 📖 贡献流程

1. **环境搭建** - 按照 [环境搭建](/docs/contributing/setup) 指南设置开发环境
2. **选择任务** - 在 Issues 中选择你想要解决的问题
3. **创建分支** - 从 master 分支创建特性分支
4. **编写代码** - 遵循 [代码规范](/docs/contributing/code-style) 编写代码
5. **运行测试** - 按照 [测试指南](/docs/contributing/testing) 确保测试通过
6. **提交 PR** - 按照 [提交流程](/docs/contributing/pr-flow) 提交 Pull Request

## 💡 贡献建议

- **从小处开始**: 从简单的 Bug 修复或文档改进开始
- **沟通优先**: 在开始工作前先与社区讨论
- **遵循规范**: 严格遵循项目的代码风格和规范
- **编写测试**: 为新功能编写相应的测试
- **更新文档**: 代码变更时同步更新文档

## 🤝 社区支持

- **GitHub Issues**: [提交问题](https://github.com/shenjianZ/ssh-terminal/issues)
- **Pull Requests**: [提交代码](https://github.com/shenjianZ/ssh-terminal/pulls)
